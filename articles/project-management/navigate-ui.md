# Navigating the user interface

[!INCLUDE[banner](includes/banner.md)]

_**Applies To:** Project Operations Integrated with ERP, Project Operations Core_

## Overview

The main project form is separated into several tabs. Each tab represents a different level of detail within the project.

- **Summary**: Provides a description of the project and aggregates both the planned and actual project performance.

    :::image type="content" source="media/navigation7.png" alt-text="Screenshot of the Summary tab and fields.":::

- **Tasks**: Provides the details regarding the work breakdown structure represented by a grid view, board view, and a Gantt.

    :::image type="content" source="media/navigation8.png" alt-text="Screenshot of the Task tab and fields.":::

- **Team**: Provides details regarding the project participants. The assigned effort of each team member is also summarized in this view.

    :::image type="content" source="media/navigation9.png" alt-text="Screenshot of the Team tab and fields.":::

- **Resource assignments**: Provides a time-phased view of the effort for each resource on a project.

    :::image type="content" source="media/navigation10.png" alt-text="Screenshot of the Resource assignments tab and fields.":::

- **Resource reconciliation**: Provides a time-phased view of the differences between the assignments of each named resource and their bookings.

    :::image type="content" source="media/navigation11.png" alt-text="Screenshot of the Resource reconciliation tab and fields.":::

- **Estimates**: Provides a time-phased view of the cost and sales estimates of a project.

    :::image type="content" source="media/navigation12.png" alt-text="Screenshot of the Estimates tab and fields.":::

- **Tracking**: Provides a view that shows the progress of tasks in the work breakdown structure for effort, cost, and sales.

    :::image type="content" source="media/navigation13.png" alt-text="Screenshot of the Tracking tab and fields.":::

- **Sales**: Provides deep links to quotes and contracts associated with the project.

- **Expense Estimates**: Provides a grid that defines project expenses based upon organizational expense categories.

    :::image type="content" source="media/navigation14.png" alt-text="Screenshot of the Expense estimates tab and fields.":::

## Grid controls

The follow is a brief overview of the typical controls found on the various project planning tabs.

### Refresh

**Refresh**: Retrieves the latest data from the server if any changes occurred after the grid was loaded.

:::image type="content" source="media/navigation7.png" alt-text="Screenshot of the Refresh button.":::

### Group by

**Group by**: Updates the grouping of the rows in the grid to reflect either resources, roles, or categories based on the user's needs.

:::image type="content" source="media/navigation6.png" alt-text="Screenshot of the Group by button.":::

### Previous/Next

**Previous**/**Next**: Update the visible time periods on the time-phased grids.

:::image type="content" source="media/navigation2.png" alt-text="Screenshot of the Previous and Next buttons.":::

### Timescale

**Timescale**: Change the aggregation of the time-phased data between days, weeks, months, and years.

:::image type="content" source="media/navigation3.png" alt-text="Screenshot of the Timescale button.":::

### Expand

**Expand**: Renders the visible grid to full screen, so you can see more roles.

:::image type="content" source="media/navigation4.png" alt-text="Screenshot of the Expand button.":::

### Time-phase by

**Time-phase by**: Updates the grouping of the rows in the grid to reflect cost estimates for sales estimates. This control also applies to the estimate script and the tracking grid.

:::image type="content" source="media/navigation0.png" alt-text="Screenshot of the Time-phase by button.":::

### Add column

**Add column**: Lets you define the visible columns in the grid. You can only add out-of-the-box columns to the grids in the **Project Planning** form.

:::image type="content" source="media/navigation5.png" alt-text="Screenshot of the Add column button.":::
